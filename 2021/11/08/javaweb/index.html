Maven

基本命令

  1. mvn compile ： 编译
  2. mvn clean ： 清理
  3. mvn test ：测试
  4. mvn package ： 打包
  5. mvn install：安装

maven生命周期

  • Maven构建生命周期描述的是一次构建过程经历的事件。
  • Maven对项目构建的 生命周期划分为三套。
    1. clean ： 清理工作
      • pre-clean
      • clean
      • post-clean
    2. default ： 核心工作：编译 测试 打包 安装
      • compile
      • test
      • package
      • install
    3. site ： 产生报告，发布站点
      • pre-site
      • site
      • post-site
  • 同一生命周期内，执行后面的命令，前面的所有命令会自动执行。

maven坐标

Maven中坐标是资源的唯一标识。

使用坐标来定义项目或者引入项目中所需要的依赖

Maven坐标的主要组成：

  • groupld：定义当前maven项目隶属组织名称
  • artifactld：定义当前Maven项目名称(通常模块名称)
  • version：定义当前项目版本号

坐标依赖

配置文件中使用<dependencies>标签选定要导入的jar包的坐标

jar包的具体坐标使用<dependency>包裹，可以添加多个jar包

    <groupId>org.example</groupId>
    <artifactId>mavendemo</artifactId>

### sql配置文件

> sql配置文件中书写需要运行的sql语句。
>
> 执行sql语句需要使用namespace属性和具体sql语句的id属性进行定位。
>
> 查询处理出的数据，想要自动封装成pojo类，需要在pojo类中的属性名和数据库名相同，如果不相同会对应不上，pojo类的属性将会是初始值null。(使用包装类)
>
> 如果pojo类中采取驼峰命名法，而数据库表中采取下划线分割法，需要在查询语句时使用别名来一一对应。
>
> 如果使用Mapper代理的方式，则namespace的值需要和定义的接口路径相同。

    <select id="selectAll" resultType="Lzy">
      select * from lzy;
    </select>
    <select id="selectId" resultType="com.doubao.pojo.Lzy">
        select id from lzy;
    </select>


</mapper>

参数占位符

可以在result映射文件中使用参数占位符，来在运行时动态的传递参数。

  1. #{} : 将其替换为 ？ ，为了防止sql注入问题。
  2. ${} : 直接进行sql语句的拼接，会存在sql注入问题。
    • 参数传递时使用#{}防止sql注入问题。
    • 表名或者列名不固定的时候使用 ${}，但是会产生sql注入问题。
  3. 使用参数占位符时，需要在对应的同名接口中使用对应的参数。
<select id="selectById" resultType="com.doubao.pojo.Lzy">

        select * from lzy where id = #{id};
</select>
public interface UserMapper{

    Lzy selectById(int id);
}

条件查询

当sql语句需要接收多个参数时，需要使用方法定义每个占位符需要传递的参数。

  1. 散装参数传递
  2. 对象传递参数
  3. map集合传递参数

多条件查询需要注意sql占位符的名称和指定使用方法需要的名称一一对应。

  • 需要传递三个参数
<select id="selectByCondition" resultType="com.doubao.pojo.Lzy">
    select * from
    where id = #{id}
    and name like #{name}
    and tal like #{tal}

</select>
散装参数传递

在对应接口的形参传递时，使用@Param注解标注需要传递到的sql语句的占位符名

@Param("sql中占位符的名字") 

List<Lzy> selectByCondition(@Param("id") int id, @Param("name") String name,@Param("tal") int tal);
对象参数

将查询的条件封装成一个对象传递到接口中，对象的属性名需要和sql语句中的占位符名相同

//    List<Lzy> selectByCondition(@Param("id") int id,@Param("name") String name);
List<Lzy> selectByCondition(Lzy lzy);
Lzy lzy = new Lzy();
lzy.setId(3);
lzy.setName("%lz%");
        List<Lzy> lzies = userMapper.selectByCondition(lzy);
map集合传递参数

将需要传递的参数值封装到map集合中，传递到对应的接口方法中，map集合中的key值需要和对应sql语句中的占位符的名称一一对应。

Map map = new HashMap();
map.put("id","7");

多条件-动态条件查询

可以在sql语句标签中使用选择标签，来动态的生成sql语句。

语法：

​ <if test=”逻辑表达式”> sql语句内容 </if>

  • 如果if标签中的逻辑表达式成立才会将会标签包裹的sql语句部分拼接到总体上，如果不成立就隐藏。
  • 逻辑表达式使用and 或者 or连接。
<select id="selectByCondition"resultType="com.doubao.pojo.Lzy">
        select * from lzy
  		where  恒等式
        <if test="id!=null">
            and id &lt; #{id}
        </if>
        <if test="name!=null and name != '' "  >
            and name like #{name}
        </if>
</select>

当使用动态条件语句拼接的时候，可能会在where关键字后拼接上and关键字，导致运行错误。

  1. 在where后拼接恒等语句

    select * from lzy
  		where  1 = 1
        <if test="id!=null">
            and sql语句
        </if>
        <if test="id!=null">
            and sql语句
        </if>

  2. where标签

    • 使用where标签代替where关键字，不需要手动拼接and等连接关键字。
    <select id="selectByCondition" resultType="com.doubao.pojo.Lzy">
        select * from lzy
        <where>
            <if test="id!=null">
                id &lt; #{id}
            </if>
            <if test="name!=null and name != '' "  >
                name like #{name}
            </if>
        </where>

    </select>

单条件-动态查询

可以从多个选择标签中选择一个拼接到sql’语句中。

<choose>标签 指定选择的范围

<otherwise>标签 如果没有成立项则执行此标签的内容

<when>标签 根据test属性的逻辑表达式选择是否执行，只会选择其中的一个，如果都不成立运行otherwise标签

<select id="selectByChoose" resultType="com.doubao.pojo.Lzy">
        select * from lzy
  		where   
        <choose>
            <when test="id != null">
                id = #{id}
            </when>
            <when test="name != null">
                `name` = #{name}
            </when>
            <when test="tal != null">
                tal = #{tal}
            </when>
            <otherwise>
                1 = 1
            </otherwise>
        </choose>
</select>
  • 如果不使用where标签自动生成where关键字，但是when中的选项没有成立，且没有书写otherwise标签，就会因为sql语句格式错误而报错。
    • 可以 使用where标签自动生成where关键字
    • 可以添加otherwise标签

添加-主键返回

添加数据时不允许手动添加主键，主键的添加方式采用自动增长，但是添加完成后无法获取当前添加的数据对应的主键值。

可以在insert标签中增加两个属性，将添加成功后自动增长后的主键值返回到实参对象中，允许程序使用。

  • useGeneratedKeys=”true”
  • keyProperty=”id” //指定返回主键存储的属性名
<insert id="add" useGeneratedKeys="true" keyProperty="id">
        insert into lzy
        values(#{id},#{name},#{tal},#{like})
</insert>

Mybatis参数传递

MyBatis接口方法中可以接收各种各样的参数，MyBatis底层对于这些参数使用不同的封装处理方式。

单个参数：

  • POJO类型
    • 可以直接使用，属性名和参数占位符的名称一致
  • Map集合
    • 可以直接使用，键名和参数占位符的名称一致
  • Collection
    • map.put(“arg0”,collection集合);
    • map.put(“collection”,collection集合);
    • 可以使用@param(“key”)注解，替换默认的arg0键名
  • List
    • map.put(“arg0”,list集合);
    • map.put(“collection”,list集合);
    • map.put(“list”,list集合);
    • 可以使用@param(“key”)注解，替换默认的arg0键名
  • Array
    • map.put(“arg0”,数组);
    • map.put(“array”,数组);
    • 可以使用@param(“key”)注解，替换默认的arg0键名
  • 其他类型：直接使用，名字无需对应

多个参数： 封装为map集合

  • map集合默认的key名字为arg0…… 和 param0……，每个传入的参数会对应着两个key名
  • 如果使用@param(“key”)注解，会将对应的key名为arg0的替换为指定key值，但是param0不会改变，还是一个参数会被保存两遍。

Mapper代理

使用Mapper代理条件

  1. 定义sql映射文件同名的Mapper接口，并且将Mapper接口和sql映射文件放置在同一目录下。
  2. 设置sql映射文件的namespace属性为Mapper接口全限定名
  3. 在Mapper接口中定义方法，方法名就是sql映射文件sql语句的id，并保持参数类型和返回值类型一致。
  4. 编码：
    1. 通过sqlSession的getMapper方法获取Mapper接口的代理对象。
    2. 调用对应方法完成sql的执行。

public static void main(String[] args) throws IOException {
        //1.加载mybatis的核心配置文件，获取sqlsessionfactory
        String resource = "mybatis-config.xml";
        InputStream inputStream = Resources.getResourceAsStream(resource);
        SqlSessionFactory sqlSessionFactory = new SqlSessionFactoryBuilder().build(inputStream);


        //2.获取sqlsession对象
        SqlSession sqlSession = sqlSessionFactory.openSession();

        //3.获取对应的Mapper代理对象
        UserMapper userMapper = sqlSession.getMapper(UserMapper.class);

        List<Lzy> lzies = userMapper.selectAll();

        System.out.println(lzies);

        //4.释放资源
        sqlSession.close();
    }

  • 当调用UserMapper接口的对应方法时，会找到与指定接口同名包下的sql映射文件，执行与执行方法名相同id的语句。

  • 如果采用Mapper代理的方式，加载sql映射文件可以简化使用包扫描的方式：

    <mappers>

<!--        <mapper resource="com/doubao/mapper/UserMapper.xml"/>-->
        
        <package name="com.doubao.mapper"/>
    </mappers>s

environment环境信息

environments用于配置数据库连接环境信息，可以配置多个不同id属性的<environment>标签，通过<environments>标签的 default属性进行切换。

<environments default="development">
       <environment id="development">
           <transactionManager type="JDBC"/>
           <dataSource type="POOLED">
               <property name="driver" value="com.mysql.jdbc.Driver"/>
               <property name="url" value="jdbc:mysql://localhost:3306/db01"/>
               <property name="username" value="root"/>

HTTP 超文本传输协议

HyperText Transfer Protocol超文本传输协议，规定了浏览器和服务器之间数据传输的规则。

HTTP协议的特点：

  • 基于TCP协议，面向连接，安全。
  • 基于请求-相应模型，一次请求对应一次响应。
  • HTTP是无状态的协议，对于事务处理没有记忆能力，每次请求-相应都是独立的。
    • 缺点：多次请求间不能共享数据。java中使用会话技术来解决问题。
    • 优点：速度快。

响应状态码：

状态码第一位数字决定了不同的响应状态，有如下：

  • 1 表示消息
  • 2 表示成功
  • 3 表示重定向
  • 4 表示客户端错误
  • 5 表示服务器错误

1xx

代表请求已被接受，需要继续处理。这类响应是临时响应，只包含状态行和某些可选的响应头信息，并以空行结束

常见的有：

  • 100（客户端继续发送请求，这是临时响应）：这个临时响应是用来通知客户端它的部分请求已经被服务器接收，且仍未被拒绝。客户端应当继续发送请求的剩余部分，或者如果请求已经完成，忽略这个响应。服务器必须在请求完成后向客户端发送一个最终响应
  • 101：服务器根据客户端的请求切换协议，主要用于websocket或http2升级

2xx

代表请求已成功被服务器接收、理解、并接受

常见的有：

  • 200（成功）：请求已成功，请求所希望的响应头或数据体将随此响应返回
  • 201（已创建）：请求成功并且服务器创建了新的资源
  • 202（已创建）：服务器已经接收请求，但尚未处理
  • 203（非授权信息）：服务器已成功处理请求，但返回的信息可能来自另一来源
  • 204（无内容）：服务器成功处理请求，但没有返回任何内容
  • 205（重置内容）：服务器成功处理请求，但没有返回任何内容
  • 206（部分内容）：服务器成功处理了部分请求

3xx

表示要完成请求，需要进一步操作。通常，这些状态代码用来重定向

常见的有：

  • 300（多种选择）：针对请求，服务器可执行多种操作。服务器可根据请求者 (user agent) 选择一项操作，或提供操作列表供请求者选择
  • 301（永久移动）：请求的网页已永久移动到新位置。服务器返回此响应（对 GET 或 HEAD 请求的响应）时，会自动将请求者转到新位置
  • 302（临时移动）：服务器目前从不同位置的网页响应请求，但请求者应继续使用原有位置来进行以后的请求
  • 303（查看其他位置）：请求者应当对不同的位置使用单独的 GET 请求来检索响应时，服务器返回此代码
  • 305 （使用代理）：请求者只能使用代理访问请求的网页。如果服务器返回此响应，还表示请求者应使用代理
  • 307 （临时重定向）：服务器目前从不同位置的网页响应请求，但请求者应继续使用原有位置来进行以后的请求

4xx

代表了客户端看起来可能发生了错误，妨碍了服务器的处理

常见的有：

  • 400（错误请求）：服务器不理解请求的语法
  • 401（未授权）：请求要求身份验证。对于需要登录的网页，服务器可能返回此响应。
  • 403（禁止）：服务器拒绝请求
  • 404（未找到）：服务器找不到请求的网页
  • 405（方法禁用）：禁用请求中指定的方法
  • 406（不接受）：无法使用请求的内容特性响应请求的网页
  • 407（需要代理授权）：此状态代码与
  • 401（未授权）类似，但指定请求者应当授权使用代理
  • 408（请求超时）：服务器等候请求时发生超时

5xx

表示服务器无法完成明显有效的请求。这类状态码代表了服务器在处理请求的过程中有错误或者异常状态发生

常见的有：

  • 500（服务器内部错误）：服务器遇到错误，无法完成请求
  • 501（尚未实施）：服务器不具备完成请求的功能。例如，服务器无法识别请求方法时可能会返回此代码
  • 502（错误网关）：服务器作为网关或代理，从上游服务器收到无效响应
  • 503（服务不可用）：服务器目前无法使用（由于超载或停机维护）
  • 504（网关超时）：服务器作为网关或代理，但是没有及时从上游服务器收到请求505（HTTP 版本不受支持）：服务器不支持请求中所用的 HTTP 协议版本

Servlet

生命周期指一个对象被创建到销毁的整个过程。Servlet运行在Servlet容器(web服务器)zhong ,其生命周期由容器来管理分为四个阶段：

  1. 加载和实例化 默认情况下，当Servlet第一次被访问，由容器创建Servlet对象。

    • 可以在@WebServlet()注解中配置，改变创建实例的时机

    • ```java
      WebServlet(urlPatterns=”url” loadOnStartup=1)

        ```

   * 负整数：第一次被访问时创建Servlet实例

   * 0或者正整数：服务器启动时创建Servlet实例，数字越小，优先级越高。

2. 初始化   在Servlet实例化之后，容器将调用Serlet的init()方法初始化这个对象，完成一些如加载配置文件，创建链接等初始化操作，该方法只会调用一次。

3. 请求处理   每次请求Servlet时Servlet容器都会调用Servlet实例的service方法，对请求进行处理。

4. 服务终止  当需要释放内存或者关闭容器时，容器就会调用Serlet的destroy()方法完成资源的释放，在destroy()方法调用之后，容器会释放这个Servlet实例，该实例会被java垃圾收集器所回收。

常用用方法：

  1. 初始化方法，在Servlet被创建时，只执行一次

    void init(ServletConfig con )

  2. 提供服务方法，每次Servlet被访问，都会调用该方法

    void service(ServletRequset req,ServletResponse res)

  3. 销毁方法，当Servlet被销毁时，调用该方法，释放内存或者关闭服务器时销毁Servlet

    public void destroy()

  4. 获取ServletConfig对象

    ServletConfig getServletConfig()

  5. 获取Servlet信息

    String getServletInfo()

Servlet urlPattern配置

Servlet想要被访问，必须配置其访问路径(urlPattern)

  • 一个servlet可以配置多个urlPattern

    @WebServlet（urlPattern={"/dome1","/dome2"}）

  • urlPattern配置规则

    • 精确匹配
      • 配置路径 @WebServlet(“/user/select”)
      • 访问路径 http://localhost:8080/TamcatDemo_war/user/select
    • 目录匹配
      • 配置路径 @WebServlet(“/user/*”)
      • 访问路径 http://localhost:8080/TamcatDemo_war/user/aaa
      • 访问路径 http://localhost:8080/TamcatDemo_war/user/bbb
    • 扩展名匹配
      • 配置路径 @WebServlet(“*.do”)
      • 访问路径 http://localhost:8080/TamcatDemo_war/select.do
      • 如果配置路径时添加正斜杠，开启服务器会报错
    • 任意匹配
      • 配置路径 @WebServlet(“/“) @WebServlet(“/*”)
      • 访问路径 http://localhost:8080/TamcatDemo_war/*

  • / 与 /* 的区别

    • 当servlet中配置了 / ，会覆盖tomcat中DefaultServlet，当其他的urlPattern都匹配不上时会运行这个servlet。
    • /* 表示匹配任意的访问结果

  • 优先级

    • 精确路径 > 目录路径 > 扩展名路径 > /* > /

解决乱码问题

乱码问题就是浏览器在传到服务器时，将数据使用UTF-8字符集来编码，服务器读取数据时，使用ISO-8859-1字符集进行解码。

  • Tomcat8版本后默认为UTF-8编码方式

post乱码问题：

protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
  //设置输入流的编码
        request.setCharacterEncoding("utf-8");
        String[] names = request.getParameterValues("name");
    }

get乱码问题：

protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        request.setCharacterEncoding("utf-8");
        String name = request.getParameter("name");
        //将浏览器获得的数据转换为二进制
        byte[] bytes = name.getBytes("ISO-8859-1");
        //将获得的二进制数组重新转换为字符串
        String str = new String(bytes,"UTF-8");


    }

通用解决方式

name = new String(name.getBytes(StandardCharsets.ISO_8859_1),StandardCharsets.UTF_8);

请求转发

请求转发(forword)：在服务器内部的资源跳转方式

实现请求转发：

request.getRequestDispatcher("资源路径").forward(request,response);

请求转发资源间共享数据：

  • void setAttribute(String name,Object o); 存储数据到requese中
  • Object getAttribute(tring name); 根据key获取值
  • void removeAttribute(String name); 根据key删除键值对

第一次请求的资源时，进行处理，将处理的结果封装到request对象中，传递给被请求转发的对象中进行下一步的处理。

请求转发特点

  • 浏览器地址栏不发生变化。
  • 只能转发到当前服务器的内部资源。
  • 一次请求，可以在多个资源间共享数据。

重定向

重定向，一种资源跳转方式

实现步骤：

  1. response.setStatus(302); 设置重定向状态码
  2. response.setHeader(“location”,”资源的虚拟路径”); 传回重定向的资源地址

简略的实现

response.sendRedirect("虚拟路径");

重定向的特点

  • 浏览器的地址栏路径发生变化
  • 可以重定向到任意位置的资源(服务器内部，外部均可)
  • 两次请求，不能在多个资源之间使用request共享数据

Response 响应数据

字符数据：
  1. 设置浏览器解析类型和Response编码方式

  2. 通过response对象获取字符输出流
    • PrintWriter writer = response.getWriter();

  • 中文数据乱码，因为response获取字符输出流的默认编码为ISO-8859-1，但是浏览器解析的编码方式为UTF-8

JSP标准标签库，使用婊气啊取代jsp页面上的java代码。

使用JSTL标签步骤

  1. 导入坐标

    • ```xml jstl jstl 1.2 taglibs standard 1.1.2 
        ```

2. 在JSP页面上引入JSTL标签库

   * ```jsp
  <%@taglib prefix="c" uri="http://java.sun.com/jsp/jstl/core" %>

  2. 在页面中使用标签

out 标签

  • 输出常量。

    可以在value属性中直接赋值。

  • 输出变量

    变量不存在的时候可以配合default属性输出默认值，
    还可以通过escapeXml控制转义字符的输出格式；
    var设定存放值的变量。

<%--输出常量(value直接赋值)--%>
<c:out value="我是out输出标签"></c:out>

<%--输出变量(value可从作用域中获取值)--%>
<c:out value="${sessionScope.name}"></c:out>

set标签

可以将值以变量的形式存放在指定的范围。

  • scope设定存放值的范围；
  • value指定存放值的内容；
  • var设定存放的变量。
<c:set scope="session" value="李四" var="name"></c:set>

remove标签

移除指定scope作用域内的变量

  • var属性是必选,指定要移除的变量名
  • scope属性是必选，指定要移除变量名的作用域
  • 如果不同的scope中存在一样的变量可以通过scope属性指定删除哪个scope中的变量

if标签

<c:if Test=”条件判断语句”></c:if>

  • 当条件判断语句成立时，if标签中的内容才会显示。
  • 条件判断语句中可以使用EL表达式
<body>
    <c:if test="${date} == 1">
        <h1>判断正确</h1>
    </c:if>
    <c:if test="${date} != 1">
        <h1>判断失败</h1>
    </c:if>
</body>

forEach标签

<c:forEach items=”遍历容器” var=“子元素”></c:dorEach>

  • items 被遍历的容器
  • var 表示产生的临时变量
  • varStatus 表示遍历的索引
    • status.index 从0开始
    • status.count 从1开始
protected void doPost(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException {
        String[] User = {"张三","李四","王五"};
        request.setAttribute("name","开始的开始");
        request.setAttribute("date",1314);
        request.setAttribute("Users",User);        request.getRequestDispatcher("/JspDemo1.jsp").forward(request,response);
    }
<c:forEach items="${User}" var="user" varStatus="status">
    <h3>${status.index} 用户名</h3> ${user}
</c:forEach>

<c:forEach begin=”0” end=”10” step=”1” var=”i”></c:dorEach>

  • begin 表示开始的索引
  • end 表示结束的索引
  • step 表示步进值
  • var 标识每次遍历的值
<c:forEach begin="0" end="10" step="1" varStatus="status" >
    第 ${status.count} 次遍历
</c:forEach>

catch标签

用来实现分支条件的控制。

  • test属性用来存放判断条件，一般使用EL表达式来编写。
  • var属性指定名称用来存放判断结果类型为true还是false。
  • scope属性用来存放var属性存放的范围。
<!--catch标签  -->
<c:catch var="error">
    <c:set target="user" property="name" value="王五"></c:set>
</c:catch>

获取catch标签中的错误信息：<c:out value="${error}"></c:out><br>

choose、when、otherwise标签

<c:choose>标签嵌套在<c:when>和<c:otherwise>标签的外面作为父标签来使用。
<c:choose>标签和<c:when>标签也可以组合使用。

<c:set var="score" value="99" scope="request"></c:set>
<c:choose>
    <c:when test="${score>90 and score<=100}">
        优秀
    </c:when>
    <c:when test="${score>80 and score<=90}">
        良好
    </c:when>
    <c:when test="${score>70 and score<=80}">
        一般
    </c:when>
    <c:when test="${score>60 and score<=70}">
        及格
    </c:when>
    <c:otherwise>
        不及格
    </c:otherwise>
</c:choose>

MVC模式

MVC是一种分层开发的模式：
M：model，业务模型，处理业务。

V：View ，视图，页面展示。

C: Controller，控制器，处理请求，调用模型和视图。

三层架构

数据访问层：对数据库的CRUD进行基本操作

​ 常用包名：web contoller

业务逻辑层：对业务逻辑进行封装，组合数据访问层中的基本功能，形成复杂的业务逻辑功能。

​ 常用包名：service

表现层：接收请求，封装数据，调用业务逻辑层，响应数据。

​ 常用包名： dao mapper
