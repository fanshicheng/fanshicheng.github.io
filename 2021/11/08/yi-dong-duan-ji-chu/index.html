移动端基础

一. 移动端基础

  • 移动端浏览器主要针对webkit内核进行兼容。
  • 开发的移动端主要针对手机端的开发。
  • 移动端碎片化严重，分辨率和屏幕尺寸大小不一样。
  • 可以使用浏览器模拟手机界面进行调试。

二. 视口

视口就是浏览器显示页面内容的屏幕区域，视口可以分为：布局视口，视觉视口，理想视口。

2.1 布局视口 layout vierport

  • 一般移动设备的浏览器都默认设置了一个布局视口，用于解决早期的pc端页面再手机上显示的问题。
  • ios,Android基本都将这个视口分辨率设置为980px，所以pc端的网页大多数都可以在手机上显示，只不过元素看上去很小，一般默认可以通过手动缩放网页。

2.2 视觉视口 visual viewport

  • 字面意思，是用户正在看到的网站的区域。 注意：是网站的区域。
  • 可以通过缩放去操作视觉视口，但是不会影响布局视口，布局视口仍保持原来的宽度。

2.3 理想视口 ideal viewport

  • 为了使网站在移动端有最理想的浏览和阅读宽度而设定。
  • 理想视口。对于设备来说，使最理想的视口尺寸。
  • 需要手动添加meta视口标签通知浏览器操作。
  • meta视口标签的主要目的：布局视口的宽度应该和理想视口的宽度一致，简单理解就是设备有多宽，布局的视口就有多宽。

2.4 视口的meta标签设置

  • 视口宽的和设备保持一致。
  • 是扣的默认缩放比例为1.0。
  • 不允许用户自行缩放。
  • 最大允许的缩放比例为1.0。
  • 最小允许的缩放比例为1.0。

	<!--设置宽度=理想宽度，不允许用户缩放，初始缩放=1.0，最大缩放=1.0，最小缩放=1.0-->
<meta name="viewport" content="width=device-width ,user-scalable=no,initial-scale=1.0,maxumun-scale=1.0,minmum-scale=1.0">
属性 说明 备注
width 宽度设置的是viewport宽度，可以设置device-width特殊值。(理想视口）
initial-scale 初始缩放比。大于0的数字。
maximum-scale 最大缩放比，大于0的数字。
minimum-scale 最小缩放比，大于0的数字。
user-scalable 用户是否可以缩放。yes或者no(1,0)

三. 二倍图

3.1 物理像素&物理像素比

  • 物理像素点指的是屏幕现实的最小颗粒。是物理真实存在的。
  • 物理像素就时分辨率。
  • 开发时的1px不是一定等于1个物理像素的
  • pc端页面，1个px等于1个物理像素，（待考证）但是移动端就不尽相同。
  • 一个px的能显示的物理像素点的个数，称为物理像素比或者屏幕像素比。

3.2 多倍图

  • 对于一张50px*50px的图片，在手机Retina屏中打开，按照物理像素比会放大倍数，造成图片的模糊。
  • 在标准的viewport设置中，使用倍图来提高图片的质量，解决在高清设备中的模糊问题。
  • 通常使用二倍图，因为苹果手机的影响，现在还存在3.4倍图的情况。
  • 背景图片注意缩放问题。
img {
    /*原始图片100px*100px*/
    width： 50px；
    height： 50px；
}
.box {
    /*原始图片100px*100px*/
    background-size:50px 50px;
}

四.移动端开发选择

4.1 单独移动端页面(主流)

通常情况下，网址域名前面加m(mobile)可以打开移动端，通过判断设备，如果是移动端打开网页，则会跳转到移动端。

4.2 响应式兼容pc移动端

通过判断屏幕宽度来改变样式，以适应不同的终端。

缺点：制作麻烦，需要话得很大的精力去调整兼容性问题。

六，移动端常见布局

移动端布局和pc端布局有所区别

先选择开发技术选型 -》 再选择布局方式

1.单独制作移动端页面

  • 流式布局(百分比布局)
  • flex弹性布局(较好)
  • less+rem+媒体查询布局
  • 混合布局

2.响应式页面兼容移动端

  • 媒体查询

流式布局

  • 流式布局，就是百分比布局，也称为非固定像素布局。
  • 通过盒子的宽度设置成百分比来根据屏幕的宽度来进行伸缩，不受固定像素的限制，内容向两侧填充。
  • 流式布局方式时移动web开发使用比较常见的布局方式。
  • max-width 最大宽度(max-height 最大高度)
  • min-width 最小宽度(min-height 最小高度)

常用初始化样式：

body {
    width: 100%;  /*流式布局*/
    min-width: 320px;   /*设置最小宽度*/
    max-width: 640px;   /*设置最大宽度*/
    margin: 0 auto;      /*设置居中*/
    font-size: 14px;     /*设置字号*/
    font-family: -apple-system,Helvetica,sans-serif;  /*设置字号*/
    color: #666;
    line-height: 1.5;   /*行高*/
}

flex弹性布局

  • 操作方便，布局极为简单，移动端应用很广泛。
  • pc端浏览器支持情况较差。
  • IE11或低版本不支持或仅部分支持。
  • 使用display:flex；

flex原理

flex意为弹性布局，为盒状模型提供最大的灵活性，任何一个容器都可以指定为flex布局。

采用flex布局的元素，称为flex容器，简称容器，他的所有子元素自动成为容器的成员，称为flex项目，简称”项目”。

通过对符合子添加flex属性，来控制子盒子的位置和排列方式

子容器可以横向排列也可以纵向排列。

  • 当为父盒子设为flex布局以后，子元素的float,clear和vertical-align属性将会失效。
  • 伸缩布局 = 弹性布局 = 伸缩盒子布局 = 弹性盒布局 = flex布局

flex常见的父项属性

以下6个属性是对父元素设置的：

属性 说明 备注
flex-direction 设置主轴的方向
justify-content 设置主轴上的子元素的排列顺序
flex-wrap 设置子元素是否换行
align-content 设置侧轴上的子元素的排列方式(多行)
align-items 设置侧轴上的子元素排列方式(单行)
flex-flow 复合属性，相当于同时设置了flex-direction和flex-wrap

flex-direction设置主轴方向

主轴和侧轴的方向

在flex布局中，分为主轴和侧轴两个方向，还可以叫做行和列，X和Y轴

  • 默认主轴方向就是X轴方向，水平向右。
  • 默认侧轴方向就是Y轴方向，水平向下。

元素是按照主轴的方向排列的，可以设置主轴的方向，剩下的就是侧轴。

属性值 说明 备注
row 从左到右 默认值
row-reverse 从右到左
column 从上到下
column-reverse 从下到上 
父盒子 {
    flex-direction: row-reverse;     //设置为从右到左排列
}

justify-content 设置主轴上的子元素排列顺序

justify-content属性定义了项目在主轴上的对齐方式。

注： 使用此属性时一定要确定好主轴的方向。

属性 说明 备注
flex-start 从头部开始，如果主轴时X轴这从左到右 默认值
flex-end 从尾部开始排列
center 从主轴居中对齐，如果主轴是X轴则水平对齐
space-around 平分剩余空间
space-between 先两边贴边，在平分剩余空间 常用

flew-warp设置子元素是否换行

默认情况下，项目都排在一条线(又称”轴线”)上。flex-wrap属性定义，flex布局中默认是不换行的。

属性值 说明
nowrap 默认值不换行
wrap 换行

align-items（单行）

该属性是控制子项在侧轴(默认是Y轴)上的排列方式在子项为单项的时候使用。

属性值 说明 备注
flex-start 从上到下
flex-end 从下到上
center 垂直居中
stretch 拉伸 需要去除拉伸方向的高度

align-content（多行）

设置子项在侧轴上的排列顺序，并且只能用于子项出现换行的情况(多行)，在单行下是没有效果的。

属性 说明 备注
flex-start 从侧轴的头部开始 默认值
flex-end 从侧轴的尾部开始
center 从侧轴的中间显示
space-around 在想在侧轴平分剩余空间
space-between 子项在侧轴先分布在两边，在评分剩余空间
stretch 设置子项元素高度评分父元素高度

flex-flow

此属性是flew-direction和flew-wrap属性的复合属性

父元素 {
    flew-flow:  row  wrap;
}

flex布局常见的子属性：

1. flex 定义子元素剩余空间

flex属性定义子项目分配剩余空间，子项目占有多少份。

默认分配的是主轴方向的份数。

可以写百分比。

可以在同一个父盒子中给多个子盒子分配不同的份额。

语法：

​ 子盒子 {

​ flex：子盒子占父盒子剩余空间的份数。

​ }

父盒子 {
    display: flex;       /*设置布局方式为弹性布局*/
}

子盒子1 {
    flex：1；       /*占父盒子剩余空间的一份份额*/
}

子盒子2 {
    flex：2；       /*占父盒子剩余空间的二份份额*/
}

子盒子3 {
    flex：20%；       /*占父盒子剩余空间的20%*/
}

2. align-self 控制子项自己在侧轴上的排列方式

align-self属性允许单个项目和其他项目不一样的对齐方式，可以覆盖align-items属性，默认值为auto，表示继承自元素的align-items属性，如果没有父元素，则等于stretch。

父盒子 {
    display: flex;       /*设置布局方式为弹性布局*/
}

子盒子 {
    align-self: flex-end；       /*设置 单独子元素侧轴的排列顺序*/
}

3. order 定义项目排列顺序

数值越小,排列越靠前,默认为0;

注意: 和z-index属性相反.

父盒子 {
    display: flex;       /*设置布局方式为弹性布局*/
}

子盒子1 {
    order:1；       /*设置 单独子元素排列顺序-向后*/
}

子盒子2 {
    order:-1；       /*设置 单独子元素排列顺序-提前*/
}

rem适配布局

可以使布局文字随着屏幕大小变化而变化。

可以跟随页面的大小变化改变高度。

可以让屏幕发生变化的时候元素高度和宽度等比例缩放。

实现原理：

​ 设置页面的媒体查询 –》在媒体查询中设置html的字体大小

​ –》 设置宽度高度为rem单位 –》当改变屏幕宽度时，媒体查询改变，html的字体大小改变，页面rem单位相应改变。

技术方案1：

​ less + 媒体查询 + rem

技术方案2：

​ flexible.js + rem

em单位

em 相对长度单位，相对于当前对象的父元素内文本的字体尺寸。

父元素字体尺寸 = em；

父元素 {
    font-size: 12px;
}

子元素 {
    height: 10em;   /*设置子元素高度为父元素字体的 10 倍，即120px*/
}

rem单位

rem (root em)是一个相对单位。类似于em，em是父元素字体大小。

不同的是rem的基准是相对于html元素的字体大小。

html字体大小 = 1 rem；

  • 与父元素字体大小无关。

优点：

​ 可以通过修改html里面的文字大小来改变页面中元素的大小，可以整体控制。

html {
    font-size: 12px;
}

子元素 {
    height: 10rem;   /*设置子元素高度为html根元素字体的10倍，即10px*/
}

媒体查询

媒体查询(media query) 是css3新语法。

使用@media查询，可以针对不同的媒体类型定义不同的样式。

@media可以针对不同的屏幕尺寸设置不同的样式。

​ 重置浏览器大小的过程中，页面也会根据 浏览器的宽度和高度重新渲染页面。

准对手机，平板等设备使用较多的媒体查询。

@media mediatype and|not|only (media feature) {
    css-Code;
}

例子： 
/*当运行在屏幕，且屏幕最大宽度为800像素时(小于等于800px)，运行。*/
@media screen and (max-width：800px) {
    body {
        background-color: green;
    }
}
/*多个并列属性--页面宽度大于800px且小于1000px*/
@media screen and (max-width：800px)  and (min-width：1000px){
    body {
        background-color: green;
    }
}
  • 用@media开头
  • mediatype 媒体类型
  • 关键字 and not only
  • media feature 媒体特性必须小括号包含

mediatype 查询类型

不同的终端设备划分成不同的类型，称为媒体类型。

值 说明 备注
all 用于所有设备
print 用于打印机和打印预览
screen 用于电脑屏幕，平板电脑，智能手机

关键字

关键字将媒体类型或多个媒体特性连接到一起作为媒体查询的条件。

  • and ：可以将多个媒体特性连接到一起，相当于’且’ 的意思。
  • not ： 排除某个指定的媒体类型，相当于’非’ 的意思。
  • only ：指定某人特定的媒体类型。可以省略。

引入资源

​ 媒体查询不仅可以单独修改某一个元素的样式表，还可以引入其他位置的样式表。实现在不同的宽度下，css样式表不同，来为一个页面两个宽度下做出较大的改变。

​ 可以在引入样式表的link标签中添加媒体查询，来实现在不同媒体查询下引入不同的样式表。

（引入标签中的if语句）

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    
    <link rel="stylesheet" href="样式表1路径" media="screen and (min-width: 320px)">
    <link rel="stylesheet" href="样式表2路径" media="screen and　（min-width:640px）">
    
    <title>Document</title>
</head>
<body>
    <section>
        <div></div>
        <div></div>
    </section>
</body>
</html>

响应式布局

​ 使用媒体查询准对不同宽度的设备进行布局和样式的设置，从而适配不同设备的目的。

​ 响应式需要一个父级作为布局容器，来配合子级元素来实现变化效果。原理：
​ 通过媒体查询来改变这个布局容器的大小，再改变里面子元素的排列顺序和大小，从而实现不同屏幕下，看到不同页面布局和样式变化。

设备划分 尺寸区间 设置宽度
超小屏幕（手机）xs <768px 100%
小屏设备（平板）sm >=768px~<992px 750px
中等屏幕（桌面显示器）md >=992px~<1200px 970px
宽屏设备（大桌面显示器）lg >=1200px 1170px

布局样式

@media screen and(min-width: 767px) {
    .container {
      width: 100%;
    }
}

@media screen and(min-width: 992px) {
  .container {
    width: 750px;
  }
}

@media screen and(min-width: 992px) {
  .container {
    width: 970px;
  }
}

@media screen and(min-width: 1200px) {
  .container {
    width: 1170px;
  }
}

七. Less基础

css是一门非程序的语言，没有变量，没有函数，作用域等概念。

  • css需要书写大量看似没有逻辑的代码，css冗余度是比较高的。
  • css没有很好的计算能力。
  • 很难写出组织良好且易于维护的css代码项目。

less介绍

Less(leaner Style Sheets) 是一门css的扩展语言，也是css的预处理器。

作为css的一种形式的扩展，它并没有减少css的功能，而是在现有的css语法上，为css加入程序式语言的特性。

在css的语法基础上引入了变量，mixin，运算以及函数等功能，大大简化css的编写，并且降低了css的维护成本，可以让我们用更少的代码做更多的事情。

新建一个后缀名为less的文件，并且在这个less文件里面书写less语句。

官网

less导入

less文件可以导入另一个less文件：

语法：

​ @import “less文件名”

可以减少html中的link标签的引入量。导入文件可以有扩展名，也可以没有扩展名。

引入的less文件和当前less文件会合并为一个css文件。

less变量

变量是指没有固定的值，可以改变的。在css中的一些颜色和数值等经常使用。

语法：

​ @变量名：值；

  • 必须以@为前缀
  • 不能包含特殊字符
  • 不能以数字为开头
  • 大小写敏感 
@color:  green;

选择器 {
    background-color: @color;
}

  • 在vscode中安装Easy Less插件，使用less文件书写相应的代码，只要保存，就回来当前路径中产生一个同名的css样式文件。

可以在一个选择器中再嵌套另一个选择器，编译器会将less代码编译为正常的css代码形式

        css代码；
    }

  • 对于两个不同的单位的值之间的运算，运算结果的值取第一个值的单位。
  • 如果两个值之间只有一个值有单位，则运算结果就取该单位。

bootstrap 框架

​ bootstrap 来自Twitter，是目前最受欢迎的前端框架。bootstrap是基于HTML，css和javascript的。简洁灵活，使得web开发更加快捷。

​ bootstrap 使用Normalize.css初始化，使用bootstrap框架时，不需要引入Normalize.css初始化。

​ 框架就是一套架构，有一套比较完整的网页功能解决方案，而且控制权在框架本身，有与之样式库，组件和插件。使用者要按照框架所规定的某种规范进行开发。

3 布局容器

​ bootstrap需要为页面内容和栅格系统包裹一个 .container 容器，bootstrap预先定义好了这个类，叫 .container

使用这些类，就不需要自己定义媒体查询，直接使用就可以了。

container类

  • 响应式布局容器，固定宽度。
  • 大屏(>=1200px) 宽度定位1170px。
  • 中屏(>=992px) 宽度定位970px。
  • 小屏(>=768px) 宽度定位750px。
  • 超小屏(100%)。

container-fluid类

  • 流式布局容器 百分百宽度。
  • 占据全部视口(viewport)的容器。
  • 适合于单独做移动端开发。

4. 栅格系统

​ 栅格系统(grid systems),可有人翻译为“网格系统”，它是指将也买你布局划分为等宽的列，然后通过列数的定义来模块化页面布局。

bootstrap 提供了一套响应式，移动设备优先的流式栅格系统，随着屏幕或者视口的尺寸的增加，系统会自动分为最多12列。

bootstrap里面container宽度式固定的，但是不同屏幕下，container的宽度不同，将container划分为12等份。

栅格系统参数

栅格系统通过一系列的行(row) 和列(column) 来组合常见页面布局。

超小屏幕(<768px) 小屏(>=768px) 中等屏(>=992px) 宽屏(>=1200px)
.container 自动(100%) 750px 970px 1170px
类前缀 .col-xs- .col-sm- .col-md- -col-lg-
列数(column) 12 12 12 12

  • 行（row）必须放到container布局容器里面。

  • 实现列的平均划分需要给列添加类前缀。

  • xs-extra small：超小 ； sm-small：小；md-medium：中等；lg-large：大；

  • 列(column)大于12.多余的 “列(column)”所在的元素将被作为一个整体另起一行排列。

  • 每一列默认有左右15像素的padding。

  • 可以同时为一列指定多个是设备的类名。以便划分不同份数：

    • class=”col-md-4 col-sm-6”

      <!--设置当前组件在大屏幕下占3份额，在中屏幕下占4份额，在小屏幕中占6份额，在超小屏幕下占12份额-->
<div class="col-lg-3 col-md-4 col-sm-6 col-xs-12"></div>

如果份数相加大于定义的总份额，多余的将会被挤到下一行显示。

如果份数相加小于定义的总份额，会占不满整个的container，会有空白。

可以使用 .col-md-offset-* 类 使得列向右侧偏移。

实际是通过*选择器为当前元素增加了左边的边距(margin)。

语法：

​ .col-md-offset-偏移的份数；

​ 其实就是为右侧盒子增加左边距

<!--设置左侧四份，右侧四份，中间偏移四份-->

    <div class="row">

    </div>

列排序

通过使用 .col-md-push-* 和 .col-md-pull-*类 就可以改变列(column)的顺序。

语法：

​ .col-md-pull-份数; //向左拉指定的份额

<div class="container">

    </div>
