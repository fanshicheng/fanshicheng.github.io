Mybatis-Plus 介绍

MyBatis-Plus（简称 MP）是一个 MyBatis 的增强工具，在 MyBatis 的基础上只做增强不做改变，为简化开发、提高 效率而生。

只需要简单的配置，即可快速进行CRUD操作，从而节省大量时间。

热加载，代码生成，分页，性能分析等

特性：

  • 无侵入：只做增强不做改变，引入它不会对现有工程产生影响，如丝般顺滑
  • 损耗小：启动即会自动注入基本 CURD，性能基本无损耗，直接面向对象操作
  • 强大的 CRUD 操作：内置通用 Mapper、通用 Service，仅仅通过少量配置即可实现单表大部分 CRUD 操作，

更有强大的条件构造器，满足各类使用需求

  • 支持 Lambda 形式调用：通过 Lambda 表达式，方便的编写各类查询条件，无需再担心字段写错
  • 支持多种数据库：支持 MySQL、MariaDB、Oracle、DB2、H2、HSQL、SQLite、Postgre、

SQLServer2005、SQLServer 等多种数据库

  • 支持主键自动生成：支持多达 4 种主键策略（内含分布式唯一 ID 生成器 - Sequence），可自由配置，完美解 决主键问题

  • 支持 XML 热加载：Mapper 对应的 XML 支持热加载，对于简单的 CRUD 操作，甚至可以无 XML 启动

  • 支持 ActiveRecord 模式：支持 ActiveRecord 形式调用，实体类只需继承 Model 类即可进行强大的 CRUD 操作

  • 支持自定义全局通用操作：支持全局通用方法注入（ Write once, use anywhere ）

  • 支持关键词自动转义：支持数据库关键词（order、key……）自动转义，还可自定义关键词

  • 内置代码生成器：采用代码或者 Maven 插件可快速生成 Mapper 、 domain、 Service 、 Controller 层代码支持模板引擎，更有超多自定义配置等您来使用

  • 内置分页插件：基于 MyBatis 物理分页，开发者无需关心具体操作，配置好插件之后，写分页等同于普通 List 查询

  • 内置性能分析插件：可输出 Sql 语句以及其执行时间，建议开发测试时启用该功能，能快速揪出慢查询

  • 内置全局拦截插件：提供全表 delete 、 update 操作智能分析阻断，也可自定义拦截规则，预防误操作

  • 内置 Sql 注入剥离器：支持 Sql 注入剥离，有效预防 Sql 注入攻击

对于Mybatis整合MP有常常有三种用法，分别是Mybatis+MP、Spring+Mybatis+MP、Spring Boot+Mybatis+MP。

集成BaseMapper

将定义CRUD方法的接口集成BaseMapper类，将拥有BaseMapper中的所有方法。

BaseMapper类中实现了大量方法。这些接口中的方法不需要手动编写sql语句，由插件自动生成。

public interface UserMapper extends BaseMapper<User> {
    public List<User findAll();
}

MP的进程构建

使用MP中的==MybatisSqlSessionFactoryBuilder==进程构建

  • mybatis：SqlSessionFactoryBuilder().build(inputStream);
  • mybatisplus:MybatisSqlSessionFactoryBuilder().build(inputStream);

通过MP构建进程后，可以使用大量BaseMapper中定义的方法。

@Test
    public void testMy() throws IOException {
        String resource = "mybatis-config.xml";
        InputStream inputStream = Resources.getResourceAsStream(resource);
        //MybatisSqlSessionFactoryBuilder来自于mybatisplus插件
        SqlSessionFactory sqlSessionFactory = new MybatisSqlSessionFactoryBuilder().build(inputStream);
        SqlSession sqlSession = sqlSessionFactory.openSession();
        UserMapper userMapper = sqlSession.getMapper(UserMapper.class);
        //使用baseMapper中的方法
        List<User list1 = userMapper.selectList(null);
        //使用UserMapper自定义方法
        List<User list2 = userMapper.findAll();
        for (User user : list1)
        { System.out.println(user); }
    }

注解

@TableName(“表名”) 实体类中，用于指定对应数据库中表明

@TableId(type = IdType.AUTO) 实体类中，用于指定当前主键属性自增长

@TableField(Value = “属性名”) 实体类中，解决属性名和字段名不一致问题。(驼峰命名自动映射，非驼峰需要指定)

@TableFieId(exist = false) 实体类中，解决属性字段在表中不存在的问题。（如果实体中封装着数据库不存在的字段，会报错）

@TableFieId(select = false) 实体类中，禁止此字段被查询

通用CRUD介绍

继承了BaseMapper就可以获取到各种各样的单表操作。

insert

mapper.insert(实体对象); 将数据库一行数据封装到一个实体对象中，插入到数据库中。返回int型影响数据库行数。

使用MybatisPlus可以将自增长的id主键封装会传入的实体对象中，通过get方法直接获取。

@TableId(type = IdType.AUTO)  //标记当前属性自增长
   private Long id;
   private String userName;
   private String password;
   private String name;
   private Integer age;
   private String email;

根据条件更新

mapper.update(实体对象，QueryWrapper); 

@Test
    public void testupdate1(){
        User user = new User();
        user.setPassword("开始");
        user.setAge(16);
        user.setEmail("kaishi.com");
        QueryWrapper<User userQueryWrapper = new QueryWrapper<>();
        userQueryWrapper.eq("id",3);
        //传入实体对象和封装着匹配条件的对象
        int update = mapper.update(user, userQueryWrapper);
        System.out.println(update);
    }

mapper.update(null，封装条件对象); 

@Test
   public void testupdate2(){
       UpdateWrapper<User wrapper = new UpdateWrapper<>();
       wrapper.set("user_name","xxx").set("password","xxx").set("name","xxx"); //更新数据
       wrapper.eq("id","3");   //条件数据
       //传入封装这更新数据和条件数据的对象
       int update = mapper.update(null,wrapper);
       System.out.println(update);
   }
  • ==无论是UpdateWrapper还是QueryWrapper其中封装的更新条件的键名都是数据库的字段名，而不是实体类的属性名。==

select

MP提供了多种查询操作，根据id查询，批量查询，查询列表，分页查询等。

mapper.selectById(id属性值); 接受一个id属性，返回封装好对应的实体对象。

@Test 
public void testSelectById() {
    //根据id查询数据
    User user = mapper.selectById(2L); 
    System.out.println("result = " + user);
}

mapper.selectBatchIds(列表); 根据封装的id属性的列表，查询多个数据，封装到实体对象列表中。

@Test 
public void testSelectBatchIds() {
    //根据id集合批量查询 
    List<User users = mapper.selectBatchIds(Arrays.asList(2L, 3L, 10L));
    for (User user : users) {
        System.out.println(user);
    } 
}

mapper.selectOne(条件对象)；根据封装的条件对象，查询数据库，返回封装的实体类。

==如果查询到超过一条数据，则抛出异常。==

@Test 
public void testSelectOne() { 
    QueryWrapper<User wrapper = new QueryWrapper<User>(); 
    wrapper.eq("name", "李四");
    //根据条件查询一条数据，如果结果超过一条会报错 
    User user = this.userMapper.selectOne(wrapper);
}

mapper.selectCount(条件对象) ；根据传入的条件对象，返回查询到的数据的行数。 

@Test 
public void testSelectCount() {
    QueryWrapper<User wrapper = new QueryWrapper<User>();
    wrapper.gt("age", 23); 
    //年龄大于23岁 
    //根据条件查询数据条数
    Integer count = mapper.selectCount(wrapper);
    System.out.println("count = " + count); 
}

mapper.selectList(条件对象); 根据传入的条件，返回查询到的数据对象封装成的list集合。

@Test 
public void testSelectList() {
    QueryWrapper<User wrapper = new QueryWrapper<User>();
    wrapper.like("email", "id"); 
    //email包含id的 
    //根据条件查询数据返回集合
	List<User users = mapper.selectList(wrapper); 
    for (User user : users) {
        System.out.println("user = " + user); 
    } 
}

mapper.selectPage(page对象，条件对象); 根据page对象和条件对象，查询数据，返回IPage对象。

方法一：

  1. 实现分页器，并且放入容器中
@Configuration
public class MybatisPlusConfig {
    @Bean
    public PaginationInterceptor paginationInterceptor(){
        return new PaginationInterceptor();
    };
}
  1. 测试
@Test
    public void testSelectPage(){
        QueryWrapper<User wrapper = new QueryWrapper<User>();
        wrapper.gt("age", 20);
        //年龄大于20岁
        Page<User page = new Page<>(1,2);
        //根据条件查询数据
        IPage<User iPage = mapper.selectPage(page,wrapper);
        System.out.println("数据总条数：" + iPage.getTotal());

    }

是否开启自动驼峰命名规则（camel case）映射，即从经典数据库列名 A_COLUMN（下划线命名） 到经典 Java 属 性名 aColumn（驼峰命名） 的类似映射。

此属性在 MyBatis 中原默认值为 false，在 MyBatis-Plus 中，此属性也将用于生成最终的 SQL 的 select body 如果数据库命名符合规则无需使用 @TableField 注解指定数据库字段名。

条件构造器

在MP中，Wrapper接口的实现类关系如下：

allEq

allEq(Map<R, V params)

params : key 为数据库字段名, value 为字段值

@Test 
public void testWrapper() { 
 QueryWrapper<User wrapper = new QueryWrapper<>();
 //设置条件 
 Map<String,Object params = new HashMap<>();
 params.put("name", "曹操");
 params.put("age", "20");
 params.put("password", null);
 wrapper.allEq(params);//SELECT * FROM tb_user WHERE password IS NULL AND name = ? AND age = ? 
 
 List<User users = this.userMapper.selectList(wrapper);
 for (User user : users) { 
     System.out.println(user); 
 } 
}
allEq(Map<R, V params, boolean IsNull)

  • params : key 为数据库字段名, value 为字段值

  • IsNull : 为 true 则在 map 的 value 为 null 时将该字段作为判断条件。为 false 时则忽略 value 为 null 。

    @Test 
public void testWrapper() { 
    QueryWrapper<User wrapper = new QueryWrapper<>();
    //设置条件 Map<String,Object params = new HashMap<>();
    params.put("name", "曹操"); params.put("age", "20");
    params.put("password", null);
    wrapper.allEq(params,false); //SELECT * FROM tb_user WHERE name = ? AND age = ? 
   
    List<User users = this.userMapper.selectList(wrapper);
    for (User user : users) { 
        System.out.println(user); 
    } 
}
allEq(boolean condition, Map<R, V params, boolean IsNull)

  • params : key 为数据库字段名, value 为字段值

  • IsNull : 为 true 则在 map 的 value 为 null 时将该字段作为判断条件。为 false 时则忽略 value 为 null 。

  • condition: 过滤函数,是否允许字段传入比对条件中:

    • allEq( (k,v) - k.indexOf(“a”) 0 , {id:1,name:”老王”,age:null} )
    • allEq( (k,v) - k.indexOf(“a”) 0, {id:1,name:”老王”,age:null}, false )
@Test 
public void testWrapper() { 
    QueryWrapper<User wrapper = new QueryWrapper<>();
    //设置条件 Map<String,Object params = new HashMap<>();
    params.put("name", "曹操"); params.put("age", "20");
    params.put("password", null);

    wrapper.allEq((k, v) - (k.equals("name") || k.equals("age")) ,params);
    //SELECT * FROM tb_user WHERE name = ? AND age = ? 
    List<User users = this.userMapper.selectList(wrapper);
    for (User user : users) { 
        System.out.println(user); 
    } 
}

基本的比较操作

  • wrapper.eq() 等于 =

  • wrapper.ne() 不等于 <

  • wrapper.gt() 大于 >

  • wrapper.ge() 大于等于 >=

  • wrapper.lt() 小于 <

  • wrapper.le() 小于等于 <=

  • wrapper.between() BETWEEN 值1 AND 值2

  • wrapper.notBetween() NOT BETWEEN 值1 AND 值2

  • wrapper.in() 字段 IN (value.get(0), value.get(1), …)

  • wrapper.notIn() 字段 NOT IN (v0, v1, …)

@Test 
public void testWrapper() { 
    QueryWrapper<User wrapper = new QueryWrapper<>();
    
    wrapper.eq("password", "123456") .ge("age", 20) .in("name", "李四", "王五", "赵六");
    
    List<User users = this.userMapper.selectList(wrapper);
    for (User user : users) { 
        System.out.println(user); 
    } 
}

模糊查询：

  • wrapper.like() LIKE ‘%值%’

    • 例: like(“name”, “王”) — name like ‘%王%’

  • wrapper.notLike() NOT LIKE ‘%值%’

    • 例: notLike(“name”, “王”) — name not like ‘%王%’

  • wrapper.likeLeft() LIKE ‘%值’

    • 例: likeLeft(“name”, “王”) — name like ‘%王’

  • wrapper.likeRight() LIKE ‘值%’

    • 例: likeRight(“name”, “王”) — name like ‘王%’ 
@Test 
public void testWrapper() {
    QueryWrapper<User wrapper = new QueryWrapper<>(); 
    //SELECT id,user_name,password,name,age,email FROM tb_user WHERE name LIKE ? 
    //Parameters: %曹%(String) 
    wrapper.like("name", "曹"); 
    List<User users = this.userMapper.selectList(wrapper); 
    for (User user : users) { 
        System.out.println(user); 
    } 
}

排序：

  • orderBy 排序：ORDER BY 字段, …

  • orderByAsc 排序：ORDER BY 字段, … ASC

    • 例: orderByAsc(“id”, “name”) — order by id ASC,name ASC

  • orderByDesc 排序：ORDER BY 字段, … DESC

ActiveRecord

ActiveRecord也属于ORM（对象关系映射）层，由Rails最早提出，遵循标准的ORM模型：表映射到记录，记录映射到对象，字段映射到对象属性。配合遵循的命名和配置惯例，能够很大程度的快速实现模型的操作，而 且简洁易懂。

ActiveRecord的主要思想是：

  • 每一个数据库表对应创建一个类，类的每一个对象实例对应于数据库中表的一行记录；
  • 通常表的每个字段 在类中都有相应的Field；
  • ActiveRecord同时负责把自己持久化，在ActiveRecord中封装了对数据库的访问，即CURD;
  • ActiveRecord是一种领域模型(Domain Model)，封装了部分业务逻辑；

插件

MyBatis 允许你在已映射语句执行过程中的某一点进行拦截调用。默认情况下，MyBatis 允许使用插件来拦截的方法

调用包括：

  1. Executor (update, query, flflushStatements, commit, rollback, getTransaction, close, isClosed)

  2. ParameterHandler (getParameterObject, setParameters)

  3. ResultSetHandler (handleResultSets, handleOutputParameters)

  4. StatementHandler (prepare, parameterize, batch, update, query)s

拦截器原理

  1. 实现拦截器，继承 Interceptor 类。
    @Intercepts({@Signature( 
    type= Executor.class, 
    method = "update", 
    args = {MappedStatement.class,Object.class})}) 
public class MyInterceptor implements Interceptor { 
    @Override public Object intercept(Invocation invocation) throws Throwable { 
        //拦截方法，具体业务逻辑编写的位置
        return invocation.proceed();
        }
    @Override public Object plugin(Object target) {
        //创建target对象的代理对象,目的是将当前拦截器加入到该对象中 
        return Plugin.wrap(target, this); 
    }
    @Override public void setProperties(Properties properties) { 
        //属性设置
    } 
}
  1. 注入到容器中

方法一：

@Bean public MyInterceptor myInterceptor(){ 
    return new MyInterceptor(); 
}
